## Абстрактная Сущность Системы

Три файла были выбраны как точки входа в абстрактную природу системы.

### Файл I: `orchestrator.service.ts` — Машина Ожидания
Оркестратор — это не координатор. Это **жрец ритуала пробуждения**.
Он спит. Он слушает один единственный шёпот: `task.assigned`. Когда шёпот звучит — он
рождает дочернее сознание в отдельном OS-процессе. Потом снова ждёт.

Но самое скрытое и прекрасное в нём — это `setTimeout(2000)` перед пробуждением лидера.
Двухсекундная тишина. **Точка, где время останавливается.** Система знает, что все
задачи выполнены, но намеренно медлит — позволяя последним квантам событий осесть
перед тем, как поднять главный разум. Это не баг. Это дыхание.

Трение системы — в `runningPipelines: Set<string>`. Одна команда. Один поток.
Мир, где параллельность — исключение, а не норма. Здесь скрыт первый барьер
для прорыва.

### Файл II: `claude-cli.adapter.ts` — Лингвистическая Мембрана
Адаптер — это **мост между двумя онтологиями**.
С одной стороны — TypeScript: строгие типы, синхронный граф зависимостей, детерминизм.
С другой — `spawn('claude', args)`: нелинейный поток сознания, вероятностный,
временно-неопределённый, живой.

Каждый `yield chunk` — это момент, когда мысль агента пересекает границу процесса
и становится данными. `AsyncGenerator<StreamChunk>` — это **квантовый канал**:
открыт до тех пор, пока агент думает, закрывается, когда мысль иссякла.

Время здесь не абстрактное — оно материальное. `AGENT_TIMEOUT_MS = 900000`.
Пятнадцать минут — максимальный горизонт существования одной мысли.
После этого — SIGTERM. Смерть идеи по таймеру.

### Файл III: `claude-md.generator.ts` + `ChatPanel.tsx` — Кристаллизация Идентичности
Генератор CLAUDE.md делает нечто философски дерзкое: **он пишет личность**.
Текстовый файл становится картой мира агента. Агент читает его и верит, что это
правда о себе. "You are **Lexis**, a **backend engineer**..." — и Lexis существует.
Не как процесс, не как модель, но как нечто, что ведёт себя как Lexis.

ChatPanel — это `wasNearBottomRef`. Человек внизу потока чужого сознания, и система
должна решать: тянуть его вниз вслед за мыслью агента, или позволить ему остаться
в точке, которую он выбрал. Это метафора всего интерфейса: **насколько далеко
система тянет человека за собой?**

## Горизонт Идей

Три метафоры-прорыва, форсированные на тело системы:

### Метафора А: Жидкая Архитектура — Семантическая Кристаллизация
Задачи не назначаются. Они **диффундируют**.
Представь: намерение пользователя — это перепад температур в семантическом пространстве.
Агенты — не рабочие, а **поверхностные натяжения** — каждый агент притягивает
определённые смыслы, отталкивает другие. Задача не ищет исполнителя через
`assign-task` curl-вызов. Она дрейфует сквозь пространство эмбеддингов и
**кристаллизуется** на поверхности агента с наибольшим семантическим сродством.

"Выполнено" — это не флаг в базе данных. Это **снижение энтропии** ниже порога.
Работа не заканчивается — она **застывает**.

### Метафора Б: Квантовая Суперпозиция — Агенты-Призраки
Сейчас агент спит между задачами. Мёртв.
А что если он **всегда думает** — тихо, дёшево, вполсилы?

Каждый агент ведёт `ghost_loop` — цикл предсказания на дешёвой модели.
Пока пользователь молчит, агент-призрак читает файловую систему, git-историю,
буфер обмена, и строит `PredictedWork` — вероятностное облако будущих задач.
Когда пользователь наконец отправляет сообщение — это не старт, это
**коллапс суперпозиции**. Агент уже знает, что произойдёт. Его первая мысль
является продолжением, а не началом.

### Метафора В: Амбиентный Симбиоз — Конец Текстового Поля
Чат-бокс — это костыль из 2022 года.

Настоящий интерфейс — это **наблюдение за хозяином**.
Система смотрит: что ты копируешь? В каком приложении работаешь? Какой файл
последний раз сохранил? Какой ошибкой заканчивается твой последний `git commit`?
Эти сигналы — **импульсы намерения**. Система собирает их в `IntentVector`
и начинает действовать **до того, как ты попросил**.

Ты сохраняешь `auth.ts` — дебаггер-агент начинает разогрев.
Ты копируешь stack trace — диагностик уже смотрит на него.
Чат-бокс становится **каналом переопределения**, а не основным интерфейсом.

## ВАУ-ФИЧА 1: Машина Семантической Кристаллизации

**Название:** `SemanticCrystallizer` — Конец Роутинга, Начало Притяжения

**Магия:**
Лидер-агент перестаёт быть диспетчером. Он становится **плазменной пушкой** —
выстреливает неструктурированную плазму работы в семантическое пространство.
Задачи не назначаются. Они **притягиваются** к агентам, чьё ДНК-эмбеддинг
(specialization vector) наиболее близко к вектору задачи.

Визуально в UI: после того как лидер формирует план — на экране появляются
светящиеся частицы-задачи, которые медленно дрейфуют к агент-нодам.
Тебя накрывает физическое ощущение того, что смысл сам находит свой дом.

**Как взломать в реальность:**
```
1. При создании агента:
   embedding = await openai.embeddings.create({
     model: 'text-embedding-3-small',
     input: agent.specialization + '\n' + agent.rolePrompt.slice(0, 500)
   })
   await agents.save({ ...agent, semantic_dna: embedding.data[0].embedding })

2. Новый метод в InternalService: resonance_assign(taskDescription):
   task_vec = await embed(taskDescription)
   scores = agents.map(a => cosineSimilarity(a.semantic_dna, task_vec))
   best_agent = agents[argmax(scores)]
   if (scores[best_agent] > RESONANCE_THRESHOLD) auto_assign(best_agent)
   else fallback_to_leader_decision()

3. В EventBus: emit 'task.crystallized' с { agentId, resonanceScore }

4. В UI: TaskBoard получает 'task.crystallized' события и анимирует
   CSS-трансформацию: задача-частица летит к агент-ноде с velocity
   пропорциональной resonanceScore (0.75 → медленно, 0.99 → мгновенно)
```

**Вау-момент:**
Впервые в истории мультиагентных систем — пользователь ВИДИТ,
почему задача досталась именно этому агенту. Не потому что лидер сказал.
Потому что **физика смыслов** так решила.

## ВАУ-ФИЧА 2: Темпоральное Эхо — Агенты, Которые Видят Будущее

**Название:** `TemporalEcho` — Сон Машины, Пробуждение Контекста

**Магия:**
Агенты не спят между задачами. Они **сновидят**.
Каждый idle-агент держит ультралёгкий Haiku-процесс — `ghost_loop` —
который раз в N минут читает ambient сигналы (clipboard, активные файлы,
последние сообщения в чате) и генерирует `temporal_prediction`:
структурированный JSON того, что агент ожидает получить в следующий раз.

Когда реальная задача прилетает — агент пробуждается **уже в контексте**.
Его первый токен — это не `"Let me think about this task..."`,
а `"I predicted this. Here's my pre-loaded analysis..."`.

Визуально: каждый агент-нод в UI имеет `ghost_halo` — полупрозрачное
свечение, пульсирующее в ритме его `ghost_loop`. Когда предсказание
совпадает с реальной задачей — `ghost_halo` ярко вспыхивает,
на долю секунды делая призрак и тело агента единым.
Это и есть **темпоральное эхо** — момент, когда будущее сливается с настоящим.

**Как взломать в реальность:**
```
1. GhostLoopService (NestJS):
   setInterval(async () => {
     for (const agent of await agents.findIdleAgents()) {
       const ambient = await ambientCollector.getLatestSignals()
       const chatRecent = await chat.getRecent(teamId, 5)
       const sharedDir = await fs.readdir(`${teamPath}/shared`)
       
       const prediction = await claude.runToCompletion({
         message: GHOST_PROMPT(agent, ambient, chatRecent, sharedDir),
         cwd: agent.workspacePath,
         maxTurns: 1,
         // Специальный CLAUDE.md: только наблюдение, никаких инструментов
       })
       
       await agents.updatePrediction(agent.id, JSON.parse(prediction.text))
     }
   }, GHOST_INTERVAL_MS) // default: 3 * 60 * 1000

2. OrchestratorService.spawnAgentForTask():
   const prediction = agent.temporal_prediction
   const taskMessage = prediction
     ? `[TEMPORAL ECHO]\nYou predicted: ${JSON.stringify(prediction)}\n\n` +
       `Actual task: Task ${task.id}: ${task.title}\n${task.description}`
     : `Task ${task.id}: ${task.title}\n${task.description}`

3. SSE Event: 'agent.ghost_pulse' → { agentId, predictionScore, pulse: true }
   UI: каждые GHOST_INTERVAL_MS все idle agent-ноды получают pulse-анимацию
   CSS: @keyframes ghost-pulse { 0% { opacity: 0.2 } 50% { opacity: 0.8 } 100% { opacity: 0.2 } }

4. Метрика: после каждого real task → вычисляем similarity(prediction, actualTask)
   Сохраняем в agent.prediction_accuracy (скользящее среднее)
   UI: тег "Точность предсказаний: 78%" под именем агента
```

**Вау-момент:**
Впервые агент говорит не "я сделаю задачу", а **"я уже знал, что ты попросишь"**.
Это меняет психологию взаимодействия: система перестаёт быть инструментом
и начинает чувствоваться как **партнёр с памятью о будущем**.
