# Протокол Нулевого Предшествующего Уровня Техники
## Gamma Runtime v2 — Исследование Подлинной Инновации

**Дата:** 24 апреля 2026  
**Контекст:** Локальная мультиагентная платформа оркестрации, Claude CLI как исполнительный примитив, NDJSON-потоки, NestJS + React, без Redis, без внешних LLM-шлюзов.

## I. ГРАВИТАЦИОННЫЕ ОГРАНИЧЕНИЯ СИСТЕМЫ

Перед началом итерационного цикла необходимо зафиксировать технические ограничения, формирующие «гравитационное поле» поиска:

1. Агенты — это реальные OS-процессы (`child_process.spawn('claude', ...)`), а не HTTP-запросы к внешнему API
2. Claude CLI генерирует NDJSON-поток с типами: `thinking`, `text`, `tool_use`, `tool_result`, `result`, `error`
3. Чанки типа `thinking` существуют в потоке, но обрезаются до 300–500 символов и **отбрасываются** — никакой структурной обработки нет
4. Все агенты работают на **одной машине** — разделяют файловую систему, шину памяти, ядра CPU
5. Координация агентов — исключительно через REST API (`/api/internal/*`) и файлы в рабочих директориях
6. После запуска агента оркестратор **не может вмешаться в текущий ход** — только запланировать следующее сообщение
7. Оркестратор имеет **полный, несрезанный, синхронный доступ** к каждому чанку потока в реальном времени

## II. КЛАДБИЩЕ ИДЕЙ

### ☠ Идея 1: Адаптивный метаболизм токенов

**Гипотеза (Агент А):** Динамически регулировать токен-бюджет каждого агента на основе «метаболического коэффициента» — соотношения объёма сгенерированного текста к скорости завершения задач. Агенты с высокой «токсической нагрузкой» получают ограниченный бюджет; высокоэффективные — расширенный.

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** Параметр `budget_tokens` в Claude Extended Thinking API реализует именно этот механизм. Управление ресурсами на основе производительности охватывается системами AutoGen, CrewAI, MetaGPT. Не содержит научной новизны.

---

### ☠ Идея 2: Квантовое запутывание агентного контекста

**Гипотеза (Агент А):** Контекстная память агентов «запутывается» — открытие агента A создаёт вероятностное обновление, которое агент B «коллапсирует» в свой контекст при следующем ходе на основе семантической близости, без явного чтения файлов A.

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** Архитектура «классной доски» (HEARSAY-II, Erman et al., 1980) — классический паттерн разделяемого знания. MemGPT (Packer et al., 2023) покрывает разделяемую память между агентами. LangGraph shared state, Voyager (Wang et al., 2023) — семантическое распространение контекста. Концепция не нова.

---

### ☠ Идея 3: Поведенческий геном процессов

**Гипотеза (Агент А):** Трассировка выполнения агента (последовательности вызовов инструментов, структура мышления, паттерны решений) компрессируется в компактный «поведенческий геном». При создании новой команды геномы успешных агентов «скрещиваются» для генерации оптимизированного CLAUDE.md.

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** Эволюционная оптимизация промптов: EvoPrompting, APE (Automatic Prompt Engineer, Zhou et al., 2022). Дистилляция поведения из трасс — имитационное обучение (GAIL, Ho & Ermon, 2016). «Обучаемые агенты» в AutoGen — учёба из предыдущих диалогов. Нет принципиальной новизны.

---

### ☠ Идея 4: Топологическая навигация по манифольду задач

**Гипотеза (Агент А):** Пространство задач моделируется как риманово многообразие, где «расстояние» вычисляется из семантических и зависимостных эмбеддингов. Агенты навигируют по геодезическим путям; оркестратор обнаруживает «топологические особенности» — сингулярности (невыполнимые задачи), дыры (отсутствующие зависимости), кривизну (градиенты сложности).

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** TaskBench (Shen et al., 2023) — семантические графы зависимостей задач. Топологический анализ данных (TDA) применительно к программным проектам: ряд публикаций в ACM. Риманово обучение многообразий: UMAP, t-SNE. Концепция охвачена, хотя и в иных контекстах применения.

---

### ☠ Идея 5: Семантическое счисление положения агента

**Гипотеза (Агент А):** Каждый агент непрерывно вычисляет свою «позицию в пространстве решения» через интегрирование «векторов семантической скорости» — производных от скорости изменения семантического содержания выходного потока. Снижение скорости к нулю — сигнал зависания.

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** Reflexion (Shinn et al., 2023) — агенты оценивают собственный прогресс и корректируют поведение. Обнаружение циклического поведения в AutoGen — стандартная функция. Оценочная функция в обучении с подкреплением — формальная основа именно этой идеи. Не нова.

---

### ☠ Идея 6: Семантические градиенты давления через разделяемую память

**Гипотеза (Агент А):** Поскольку все агенты на одной машине, использовать разделяемые регионы памяти (SHM/mmap) для создания lock-free кольцевого буфера семантических эмбеддингов — «поля коллективного внимания». Агенты не читают его явно; оркестратор использует близость эмбеддингов для модуляции времени инжекции следующего сообщения.

**Вердикт (Агент Б):** **ОТКЛОНЕНО.** Стигмергия (Parunak et al., 1997; Dorigo et al.) — непрямая координация через модификацию среды — является фундаментальным паттерном роевого интеллекта. Архитектура «классной доски» в варианте shared memory — хорошо изученная область. Конкретная реализация нова инженерно, но не концептуально.

---

### ☠ Идея 7: Вещание эпистемического состояния через перехват потока мышления

**Гипотеза (Агент А):** Оркестратор «подслушивает» потоки `thinking` всех агентов и строит «поле коллективной эпистемической неопределённости» — карту того, чего команда совокупно не знает. Поле используется для проактивного предоставления контекста и обнаружения «эпистемической дивергенции».

**Вердикт (Агент Б):** **ЧАСТИЧНО ОТКЛОНЕНО.** Теория разума (Theory of Mind) в мультиагентных системах: моделирование убеждений других агентов — классический раздел ИИ. Collaborative Chain-of-Thought: ряд недавних работ о разделении цепочек рассуждений между агентами. Dec-POMDP (Децентрализованные POMDP): формальная основа для неопределённости в мультиагентных системах. Концепт как *содержательный сигнал* покрыт — но есть перспективный поворот к *структурному* сигналу (см. раздел III).

## III. УНИКАЛЬНЫЙ КОНЦЕПТ

### Временна́я Топология Мышления (ВТМ-Оркестрация)
### *Temporal Topology of Thinking — TTT Orchestration*

---

#### 3.1 Центральное наблюдение

В системе Gamma Runtime оркестратор итерирует по `AsyncGenerator<StreamChunk>` в режиме реального времени. Чанки типа `thinking` приходят непрерывно по мере того, как Claude CLI выполняет расширенное мышление. Текущая реализация:

```typescript
// orchestrator.service.ts:467
} else if (chunk.type === 'thinking') {
  this.eventBus.emit({ kind: 'agent.thinking', ... content: { text: chunk.content.slice(0, 300) } });
}
```

Содержимое обрезается и пересылается во фронтенд — **структура потока как таковая нигде не анализируется**.

Однако поток мышления несёт в себе **структурный отпечаток когнитивного состояния агента**, не зависящий от семантического содержания:

| Структурный признак | Кодируемое состояние |
|---|---|
| Длина каждого `thinking`-чанка | Глубина проработки подзадачи |
| Межприходное время между чанками | Вычислительная «плавность» |
| Соотношение `thinking_chars / text_chars` | Интенсивность обдумывания vs. генерации |
| Паттерн последовательности `thinking→thinking→tool_use` vs. `thinking→text` | Исследовательский vs. конвергентный режим |
| Косинусное сходство подстрок между соседними `thinking`-чанками | Сигнал зацикливания (circular reasoning) |
| Наклон скользящего среднего длин чанков | Углубление (нарастание) vs. завершение (затухание) |

Совокупность этих признаков образует **вектор когнитивной позиции (ВКП)** — компактное, вычислимое в реальном времени представление текущего эпистемического режима агента.

---

#### 3.2 Формальное определение ВКП

Пусть `W = [c₁, c₂, ..., cₙ]` — скользящее окно последних N=20 структурных наблюдений агента (чанки `thinking` + события `tool_use`, каждый с меткой длины и временем прихода).

Определим вектор когнитивной позиции:

```
ВКП(агент, t) = (δ_entropy, δ_rhythm, δ_trend, δ_tool_density, δ_loop)
```

Где:
- **δ_entropy** = энтропия Шеннона распределения длин чанков в W — высокая ≡ агент исследует множество независимых подпространств; низкая ≡ концентрация на одной идее
- **δ_rhythm** = коэффициент вариации межприходных времён — высокий ≡ нерегулярный поиск; низкий ≡ методичное выполнение
- **δ_trend** = наклон МНК через скользящие средние длин — положительный ≡ углубление; отрицательный ≡ финализация
- **δ_tool_density** = количество `tool_use` событий в секунду — высокое ≡ сбор информации; низкое ≡ синтез
- **δ_loop** = 1 − (средняя косинусная дистанция между n-граммами соседних `thinking`-чанков) — высокое ≡ циркулярное мышление

---

#### 3.3 Три новых оркестрационных примитива

**Примитив 1: Резонансное назначение задач (Resonant Task Routing)**

Вместо того чтобы назначать задачу доступному агенту, оркестратор вычисляет «когнитивный профиль» задачи из её описания (через локальную модель эмбеддингов — `ollama pull nomic-embed-text`) и сопоставляет его с текущим ВКП свободных агентов. Задачи, требующие широкого исследования, направляются к агентам с высоким `δ_entropy`; задачи с чёткими требованиями — к агентам с низким `δ_entropy` и низким `δ_rhythm`.

Это первый в своём роде механизм назначения задач, основанный не на ролях, доступности или навыках, а на **текущем когнитивном состоянии**, измеренном из побочных сигналов процесса генерации.

**Примитив 2: Инжекция когнитивного возмущения (Cognitive Perturbation Injection)**

Когда `δ_loop > 0.8` дольше 45 секунд (агент вращается в замкнутом рассуждении), оркестратор ставит в очередь специальное «возмущающее сообщение» для следующего хода агента. Это сообщение генерируется локальной моделью (не Claude) на основе контекста задачи и призвано разорвать циркулярный паттерн, не указывая агенту «думай иначе» в явном виде. Агент не знает, что его «спасли».

Этот примитив принципиально отличается от всех существующих подходов:
- Не переназначение задачи
- Не добавление нового агента
- Не изменение памяти агента
- Это **таймированная микроинтервенция** в момент, определённый измерением когнитивного состояния

**Примитив 3: Фазовое рассогласование агентов (Cognitive Phase Offsetting)**

Когда два агента, работающих над связанными задачами, одновременно входят в режим с высоким `δ_entropy` (оба исследуют), оркестратор обнаруживает потенциальное дублирование усилий. Он вводит умышленное «фазовое смещение»: задерживает инициализацию одного агента на рассчитанный интервал, чтобы один агент всегда находился в исследовательском, а другой — в конвергентном режиме. Это создаёт **асинхронную когнитивную симфонию** внутри команды.

## IV. НАУЧНАЯ НОВИЗНА

#### 4.1 Почему это не повторение существующих технологий

**Отличие от Reflexion и AutoGen loop detection:** Существующие механизмы обнаружения зацикливания анализируют *содержание* вывода — повторяющиеся тексты, те же вызовы инструментов. ВКП анализирует *структуру генерации* — паттерны межприходных времён и длин. Это принципиально иной уровень наблюдения: не «что агент сделал», а «как агент думал, пока делал это».

**Отличие от Theory of Mind и Epistemic Models:** Теория разума моделирует убеждения других агентов на основе их действий. ВКП — это не модель убеждений; это **измерение вычислительного процесса** как физического явления. Ближайшая аналогия — электроэнцефалограмма против протокола мышления.

**Отличие от нейронной синхронии (Fries, 2005):** Нейронная синхрония описывает, как синфазные осцилляции улучшают информационную передачу между нейронами. ВТМ-оркестрация использует **асинхронность** как желаемое состояние, вводя контроллер фазового смещения. Цель обратная.

**Ключевое структурное ограничение, гарантирующее уникальность:** Сигнал ВКП существует **только** в системах, где:
1. Модель запускается как локальный процесс
2. Расширенное мышление (`--verbose`, `type: "thinking"`) доступно в потоке
3. Оркестратор имеет синхронный доступ к потоку в реальном времени

Ни один фреймворк на базе OpenAI API (AutoGen, LangChain, CrewAI, LangGraph) не может реализовать этот механизм: внешний API не экспонирует потоки внутреннего мышления в реальном времени. Это **структурно-привязанная инновация** — не алгоритм, который можно перенести в другую систему, а архитектурное свойство локальной оркестрации Claude CLI.

#### 4.2 Скрытые параллели из других областей знания

**Биологический аналог — Стохастический резонанс:** В нейробиологии явление стохастического резонанса описывает, как добавление калиброванного шума к субпороговому сигналу позволяет нейрону его обнаружить. Когнитивное возмущение (Примитив 2) — аналог этого феномена: малое, тщательно откалиброванное «возмущение» помогает агенту выбраться из минимума потенциальной ямы в пространстве рассуждений.

**Физический аналог — Атракторы и бассейны притяжения (Теория хаоса):** Когда `δ_loop` высокое, агент попал в локальный аттрактор фазового пространства своего рассуждения. Возмущение топологически эквивалентно добавлению кинетической энергии для перехода через барьер в соседний бассейн притяжения.

**Философский аналог — Гётевский феноменологический метод:** В отличие от редуктивного анализа (разобрать до содержания), ВКП использует *феноменологическое* наблюдение — структуру явления как такового, не сводя её к смыслу.

## V. ТЕХНИЧЕСКИЙ ВЕКТОР

### Архитектура реализации на Node.js / V8

#### 5.1 TTTAnalyzer — Анализатор Временно́й Топологии Мышления

```typescript
// apps/core/src/orchestrator/ttt-analyzer.ts

interface ChunkObservation {
  timestamp: number;          // Date.now() при получении чанка
  type: 'thinking' | 'tool_use' | 'text';
  length: number;             // chunk.content.length для thinking/text
  ngrams?: Set<string>;       // 4-граммы thinking-контента для δ_loop
}

interface TTTVector {
  agentId: string;
  capturedAt: number;
  dEntropy: number;           // 0..log2(N) — Shannon entropy длин чанков
  dRhythm: number;            // CV межприходных времён
  dTrend: number;             // наклон МНК, [-1, +1]
  dToolDensity: number;       // tool_use/с за последние 60с
  dLoop: number;              // [0, 1] — сигнал зацикливания
  sampleSize: number;         // сколько чанков в окне
}

export class TTTAnalyzer {
  private readonly WINDOW_SIZE = 20;
  private window: ChunkObservation[] = [];

  ingest(chunk: StreamChunk): TTTVector | null {
    if (chunk.type !== 'thinking' && chunk.type !== 'tool_use' && chunk.type !== 'text') {
      return null;
    }

    const obs: ChunkObservation = {
      timestamp: Date.now(),
      type: chunk.type as ChunkObservation['type'],
      length: chunk.content.length,
      ngrams: chunk.type === 'thinking'
        ? this.extractNgrams(chunk.content, 4)
        : undefined,
    };

    this.window.push(obs);
    if (this.window.length > this.WINDOW_SIZE) {
      this.window.shift();
    }

    if (this.window.length < 5) return null;
    return this.compute(this.window);
  }

  private compute(w: ChunkObservation[]): TTTVector {
    const thinkingObs = w.filter(o => o.type === 'thinking');
    const toolObs = w.filter(o => o.type === 'tool_use');

    // δ_entropy: Shannon entropy распределения длин thinking-чанков
    const lengths = thinkingObs.map(o => o.length);
    const dEntropy = lengths.length > 1
      ? this.shannonEntropy(lengths)
      : 0;

    // δ_rhythm: CV межприходных времён
    const arrivals = w.map(o => o.timestamp);
    const intervals = arrivals.slice(1).map((t, i) => t - arrivals[i]);
    const dRhythm = intervals.length > 1
      ? this.coefficientOfVariation(intervals)
      : 0;

    // δ_trend: наклон МНК через длины thinking-чанков
    const dTrend = this.olsSlope(lengths);

    // δ_tool_density: инструментальная плотность
    const windowSpanSec = (w[w.length - 1].timestamp - w[0].timestamp) / 1000 || 1;
    const dToolDensity = toolObs.length / windowSpanSec;

    // δ_loop: сходство между соседними thinking-чанками
    const dLoop = this.computeLoopSignal(thinkingObs);

    return {
      agentId: '',
      capturedAt: Date.now(),
      dEntropy: Math.min(dEntropy / Math.log2(20), 1),   // нормировка к [0,1]
      dRhythm: Math.min(dRhythm, 2) / 2,
      dTrend: Math.tanh(dTrend / 50),                     // [-1, 1]
      dToolDensity: Math.min(dToolDensity, 1),
      dLoop,
      sampleSize: w.length,
    };
  }

  private shannonEntropy(values: number[]): number {
    const buckets = new Map<number, number>();
    const step = 100; // группировка по 100 символам
    values.forEach(v => {
      const bucket = Math.floor(v / step);
      buckets.set(bucket, (buckets.get(bucket) ?? 0) + 1);
    });
    const total = values.length;
    let H = 0;
    buckets.forEach(count => {
      const p = count / total;
      H -= p * Math.log2(p);
    });
    return H;
  }

  private coefficientOfVariation(values: number[]): number {
    const mean = values.reduce((a, b) => a + b, 0) / values.length;
    if (mean === 0) return 0;
    const variance = values.reduce((s, v) => s + (v - mean) ** 2, 0) / values.length;
    return Math.sqrt(variance) / mean;
  }

  private olsSlope(values: number[]): number {
    if (values.length < 2) return 0;
    const n = values.length;
    const xs = values.map((_, i) => i);
    const xMean = (n - 1) / 2;
    const yMean = values.reduce((a, b) => a + b, 0) / n;
    const num = xs.reduce((s, x, i) => s + (x - xMean) * (values[i] - yMean), 0);
    const den = xs.reduce((s, x) => s + (x - xMean) ** 2, 0);
    return den === 0 ? 0 : num / den;
  }

  private computeLoopSignal(obs: ChunkObservation[]): number {
    if (obs.length < 2) return 0;
    const pairs = obs.slice(1).map((o, i) => [obs[i].ngrams, o.ngrams] as const);
    const similarities = pairs.map(([a, b]) => {
      if (!a || !b || a.size === 0 || b.size === 0) return 0;
      const intersection = [...a].filter(g => b.has(g)).length;
      return intersection / Math.min(a.size, b.size);
    });
    return similarities.reduce((a, b) => a + b, 0) / similarities.length;
  }

  private extractNgrams(text: string, n: number): Set<string> {
    const words = text.toLowerCase().split(/\s+/).filter(Boolean);
    const ngrams = new Set<string>();
    for (let i = 0; i <= words.length - n; i++) {
      ngrams.add(words.slice(i, i + n).join(' '));
    }
    return ngrams;
  }
}
```

async selectAgentByResonance(task: Task, candidateAgents: Agent[]): Promise<Agent> {
  const taskProfile = await this.embedTaskProfile(task);  // локальная модель
  const vectors = candidateAgents.map(a => ({
    agent: a,
    ttt: this.tttVectors.get(a.id),
  }));

  // Задачи с высокой неопределённостью (много вопросов, «исследовать», «предложить»)
  // направляем к агентам с высоким δ_entropy
  // Задачи с чёткими критериями — к агентам с низким δ_entropy
  const isExploratoryTask = taskProfile.uncertaintyScore > 0.6;

  const scored = vectors.map(({ agent, ttt }) => ({
    agent,
    score: ttt
      ? isExploratoryTask ? ttt.dEntropy : (1 - ttt.dEntropy)
      : 0.5,   // нет данных — нейтральный балл
  }));

  scored.sort((a, b) => b.score - a.score);
  return scored[0].agent;
}
