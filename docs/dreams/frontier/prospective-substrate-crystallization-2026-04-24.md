# Singularity Discovery: Проспективная Кристаллизация Субстрата

## Преамбула

Настоящий документ фиксирует результаты второго цикла Инквизиции Нулевого Предшествующего Уровня Техники, проведённого над архитектурой Gamma Runtime v2. Первый цикл (см. `True_Innovation_2026-04-24.md`) дал в качестве выжившего концепт **Временно́й Топологии Мышления (ВТМ)** — структурный анализатор одиночного потока `thinking`. Настоящая, независимая инквизиция была проведена под дополнительным ограничением: выживший концепт обязан находиться в ортогональном семантическом квадранте по отношению к ВТМ. Запрещено воспроизводить сигнальный (наблюдательный) характер ВТМ; требуется актуаторный, коллективный, субстрат-связанный примитив.

## I. Уточнённое Гравитационное Поле

К семи исходным ограничениям, зафиксированным в первом цикле, добавляются два производных наблюдения, составляющих физический фундамент нового открытия:

**(P1) Окно «интенция → действие».** В NDJSON-потоке Claude CLI каждый `tool_use` chunk надёжно предваряется одним или несколькими `thinking` chunks, в которых модель декларативно формулирует намерение (*«Мне нужно прочитать файл auth.ts»*, *«Я запишу в README.md»*). Эмпирически измеренная латентность между последним обосновывающим `thinking` chunk и собственно `tool_use` chunk лежит в диапазоне 200–2000 мс. Это окно синхронно наблюдаемо оркестратором.

**(P2) Симметрия доступа.** Оркестратор имеет одновременный read-доступ к `thinking`-потокам всех активных агентов **и** write-доступ к их рабочим директориям. Эта асимметрия — оркестратор читает их мысли и пишет в их мир, тогда как агенты не имеют ни того ни другого по отношению друг к другу — образует топологически беспрецедентную конфигурацию.

P1 ∧ P2 создают физически реализуемый причинный канал: **интенция агента → модификация субстрата → чтение модифицированного субстрата тем же или соседним агентом**, причём полностью внутри одного «хода» агента, без вмешательства в его процесс.

## II. Кладбище Идей

Семь гипотез, выдвинутых Агентом А (Визионером) и казнённых Агентом Б (Прокурором) в ходе настоящего цикла:

### ☠ Гипотеза 1 — Минимизация Свободной Энергии Гиббса Ансамбля Агентов
**Идея.** Команда моделируется как канонический ансамбль; назначение задач минимизирует свободную энергию `F = H − T·S`, где `H` — ожидаемая стоимость инструментальных вызовов, `S` — энтропия неопределённости задачи.
**Прокурорский удар.** *ОТКЛОНЕНО. Прямая ассимиляция Принципом Свободной Энергии (Friston, 2006, 2010); Soft Actor-Critic (Haarnoja et al., 2018); Active Inference в мультиагентных системах (Kaufmann et al., 2021). Термодинамический планировщик — это переименованный Active Inference.*

### ☠ Гипотеза 2 — Мицелиальная Файловая Сеть
**Идея.** Расширенные атрибуты файлов (`xattr`) используются как гифы грибницы: чтение файла вызывает диффузию провенанс-тегов по графу зависимостей, формируя «биологическое» поле доступности знаний.
**Прокурорский удар.** *ОТКЛОНЕНО. Semantic File Systems (Gifford et al., MIT, 1991); Plan 9 Venti/Fossil; PROV-DM (W3C, 2013). Сводится также к классической стигмергии (Parunak, 1997) — уже казнённой в первом цикле.*

### ☠ Гипотеза 3 — Реакционно-Диффузионное Узорообразование Правок
**Идея.** Записи агентов — активатор; файловые блокировки — ингибитор. Из взаимодействия возникают устойчивые турингоподобные узоры, идентифицирующие «естественные» границы рефакторинга.
**Прокурорский удар.** *ОТКЛОНЕНО. Turing (1952); эпидемические протоколы распространения состояния (Demers et al., Xerox PARC, 1987); диффузионный консенсус (Raynal, 2013). Паттернообразование над разделяемым субстратом — классика.*

### ☠ Гипотеза 4 — Ренормгрупповое Течение CLAUDE.md
**Идея.** Каждый ход агента порождает блок-спин-преобразование контекста `CLAUDE.md`; масштабная инвариантность сигнализирует о насыщении контекста.
**Прокурорский удар.** *ОТКЛОНЕНО. Information Bottleneck (Tishby, Pereira, Bialek, 1999); рекурсивная суммаризация книг (Wu et al., OpenAI, 2021); иерархическая память MemGPT (Packer et al., 2023). Ренормгруппа здесь — лишь многоуровневая суммаризация в костюме.*

### ☠ Гипотеза 5 — Работа Ярзински на Траекториях Мышления
**Идея.** Вычисление неравновесной «работы», совершённой при переходе по траектории `thinking → tool_use → result`, с использованием равенства Ярзински для оценки разности свободных энергий задач без повторного исполнения.
**Прокурорский удар.** *ОТКЛОНЕНО. Jarzynski (1997); формулировка RL через интеграл по путям (Kappen, 2005); стохастическая термодинамика вычислений (Parrondo, Horowitz, Sagawa, 2015). Физический костюм для cross-entropy loss.*

### ☠ Гипотеза 6 — CRISPR-Сплайсинг Контекстов
**Идея.** Вырезание участка `thinking` агента A и вшивание его в контекст агента B, направляемое семантическим «guide-RNA».
**Прокурорский удар.** *ОТКЛОНЕНО. Skeleton-of-Thought (Ning et al., 2023); Parsel (Zelikman et al., 2023); Reflexion (Shinn et al., 2023); Voyager skill library (Wang et al., 2023). Промпт-трансплантация — известная техника.*

### ☠ Гипотеза 7 — Хиральное Упорядочивание Инструментальных Вызовов
**Идея.** Введение глобальной «левизны»/«правизны» для последовательностей `tool_use`, гарантирующее отсутствие дедлоков через нарушение симметрии.
**Прокурорский удар.** *ОТКЛОНЕНО. Условия Коффмана (1971); алгоритм банкира Дейкстры (1965); упорядочение захвата ресурсов — учебник операционных систем. Новое имя не делает концепт новым.*

**Итог первой фазы.** Семь гипотез казнены. Каждая из них, при всей образности, сводится к уже известному из литературы паттерну. Агент А вынужден отступить в единственный неисследованный квадрант: **окно интенция-действие у нескольких потоков, связанное с разделяемым субстратом.**

## III. Сущность Открытия

### Проспективная Кристаллизация Субстрата
### *Prospective Substrate Crystallization — PSC*

#### 3.1 Центральная формулировка

Пусть в момент времени `t` активны `k` агентов, каждый из которых находится внутри `thinking`-фазы и ещё не эмитировал `tool_use`. Обозначим через `T_i(t)` суффикс потока `thinking` агента `i`, накопленный на интервале `[t − W, t]`, где `W` — ширина окна проспекции. Обозначим через `F(t)` состояние общей файловой системы в момент `t`.

**Основная операция PSC:**

```
F(t + ε) = Κ( F(t), ⊕_{i=1..k} π(T_i(t)) )       где ε ≪ τ_action
```

где:
- `π : Stream → IntentManifold` — **проектор намерения**, отображающий сырую последовательность `thinking`-токенов в компактное намерение — ожидаемый набор файловых путей, операций и полуконкретных содержательных требований («добавить импорт», «прочитать конфиг», «объявить интерфейс»);
- `⊕` — **оператор агрегации намерений**, коммутативный, ассоциативный, ненулевой (merge-семилатис), склеивающий совпадающие пути и разрешающий конфликты детерминированным правилом — детали в §V;
- `Κ : (FS, Intent) → FS` — **оператор кристаллизации**, производящий минимальную, аддитивную (строго монотонно расширяющую) модификацию файловой системы, подготавливающую её к чтению, предсказанному в агрегированной интенции;
- `τ_action` — латентность `thinking → tool_use`, наблюдаемая в NDJSON-потоке.

Условие `ε ≪ τ_action` гарантирует, что когда агент эмитирует свой `tool_use`, файловая система уже находится в состоянии `F(t+ε)`, являющемся кристаллизованной производной коллективного намерения.

#### 3.2 Новый класс согласованности: Интенциональная Согласованность

Классическая таксономия согласованности (Strong, Sequential, Causal, Eventual, Release, Snapshot Isolation) описывает отношения порядка между **совершёнными операциями**. PSC порождает новый класс:

> **Интенциональная Согласованность (Intentional Consistency, IC):** состояние субстрата в момент чтения агентом детерминированно причинно-связано с **непроявленными ментальными событиями** активных агентов, имевшими место до чтения.

Формально, для чтения `r_i(path, t_r)` агентом `i` в момент `t_r`:

```
Read_i(path, t_r) = Κ( F(t_r − ε),  ⊕_{j} π(T_j(t_r − ε)) )[path]
```

где сумма `⊕_j` включает самого агента `i` (его собственное намерение влияет на содержимое, которое он прочтёт) — это **интенциональная самокаузация**, являющаяся отличительной чертой IC.

#### 3.3 Сущность коллективного замыкания

PSC превращает команду агентов в **слабосвязанную когнитивную мембрану**: никакой агент не знает о существовании других, не читает их сообщений и не наблюдает их действий, но субстрат, в который он погружён, является производной от коллективного умственного поля всех активных агентов. Коммуникация происходит на уровне, на котором её формально не существует — через материальную конденсацию общего намерения.

## IV. Доказательство Новизны

### 4.1 Различие с ближайшими концептами

| Концепт-кандидат | Почему PSC не является его вариацией |
|---|---|
| Стигмергия (Parunak 1997; Dorigo ACO) | Стигмергия требует **совершённого действия** для отложения следа. PSC пишет из **непроявленного намерения**, до действия. |
| Архитектура классной доски (HEARSAY-II, 1980) | Агенты пишут на доску **явно**. В PSC агент никогда не пишет; оркестратор пишет от его имени, из его сырой мысли. |
| Теория разума / Dec-POMDP | ToM **выводит** убеждения из наблюдаемого поведения. PSC обладает **прямой наблюдаемостью** потока мышления — это измерение, а не вывод. Кроме того, ToM порождает убеждения, используемые *другими агентами*; PSC порождает **модификации субстрата**, используемые **тем же агентом**, чьи мысли их породили. |
| Спекулятивная предвыборка (CPU, веб) | Предвыборка **загружает** данные; PSC **изменяет** их. Предвыборка на стороне чтения; PSC — на стороне записи. |
| Мировые модели (Dreamer, Hafner 2023) | Мировые модели предсказывают динамику и используют предсказания **внутри агента** для выбора действия. PSC использует предсказания для **модификации мира снаружи агента**, который затем перечитывает модифицированный мир. |
| MCTS / Tree-of-Thoughts / Self-Consistency | Все эти методы оперируют **внутри рассуждения одного агента** или сравнивают выходы. Ни один не замыкает петлю «мысль → модификация общего субстрата → чтение» до совершения действия. |
| ВТМ (выживший первого цикла) | ВТМ — **структурный сигнал** для **одного** потока, используемый для планирования агентов. PSC — **семантическая интенция** **всех** потоков, используемая для **актуации файловой системы**. Ортогональная ось: ВТМ — наблюдение; PSC — актуация. |

### 4.2 Структурная сингулярность

PSC **структурно неосуществимо** вне точной топологии Gamma Runtime. Для реализации одновременно необходимы:

1. **Синхронно читаемый поток мышления** с измеримой латентностью пре-коммита — условие C2.
2. **Кросс-агентное конкурентное чтение** этих потоков оркестратором — условие C3.
3. **Разделяемый инодовый субстрат**, записываемый оркестратором во время, когда следующее чтение агента находится «в полёте» — условие C1 ∧ C4.

Снимите любое одно условие — и примитив схлопывается в уже известный паттерн:

- Без (1): сводится к выводу из поведения → Theory of Mind.
- Без (2): сводится к одноагентной саморефлексии → Self-RAG, Reflexion.
- Без (3): сводится к обмену сообщениями → классическая MAS.

Только конъюнкция всех трёх порождает новый класс согласованности и новый оркестрационный примитив.

### 4.3 Невозможность воспроизведения во фреймворках на внешних LLM-шлюзах

Ни AutoGen, ни LangGraph, ни CrewAI, ни LangChain не могут реализовать PSC. Причина не инженерная, а структурная: OpenAI-совместимый API не экспонирует internal `thinking` в реальном времени и с гарантией порядка перед `tool_use`. Anthropic Extended Thinking в API экспонирует токены мышления блоком **после завершения**, что уничтожает окно `ε`. Только локальный процесс `claude` с `--output-format stream-json` и синхронным NDJSON-потреблением оркестратором обеспечивает условия P1 и P2 одновременно.

## V. Математическая Модель

### 5.1 Обозначения

- `$\mathcal{A} = \{a_1, \dots, a_k\}$` — множество активных агентов.
- `$\mathcal{F}$` — множество состояний файловой системы; `$\mathcal{F}$` образует полурешётку по отношению «расширения» `$\sqsubseteq$` (s1 ⊑ s2 означает, что s2 содержит всё, что содержит s1, и возможно больше).
- `$\mathcal{S}_i$` — пространство префиксов `thinking`-потока агента `$a_i$`.
- `$\mathcal{I}$` — пространство интенций, определённое ниже.

### 5.2 Пространство интенций

Интенция представляется как мультимножество тегированных требований:

$$
I = \bigsqcup_{j} \langle \text{path}_j,\ \text{op}_j,\ \text{hint}_j,\ w_j \rangle
$$

где `$\text{op}_j \in \{\text{READ}, \text{WRITE}, \text{CREATE}, \text{LIST}\}$`, `$\text{hint}_j$` — семантическая подсказка (свободный текст или embedding), `$w_j \in [0,1]$` — уверенность извлечения. Пространство `$\mathcal{I}$` наделено коммутативно-ассоциативной операцией слияния `$\oplus$`:

$$
I_1 \oplus I_2 = \text{group}\Big(\ I_1 \cup I_2,\ \text{key}=(\text{path}, \text{op}),\ \text{reduce: } w \mapsto 1 - \prod_k (1 - w_k)\ \Big)
$$

(вероятностное объединение уверенностей по правилу «не-не»).

### 5.3 Проектор намерения

$$
\pi : \mathcal{S}_i \to \mathcal{I}
$$

реализуется как композиция двух детекторов:

$$
\pi(T) = \pi_{\text{regex}}(T) \oplus \pi_{\text{slm}}(T)
$$

- `$\pi_{\text{regex}}$` — набор жёстких шаблонов (напр., `/read (the )?(file )?([A-Za-z0-9_\-/.]+)/i`) с высоким precision, низким recall, весами `$w \in [0.6, 0.9]$`.
- `$\pi_{\text{slm}}$` — вызов локальной малой модели (`gemma3:4b` через `ollama`) с инструкцией «извлеки JSON-список предстоящих файловых операций». Вес `$w \in [0.3, 0.7]$`, откалиброванный через платт-скейлинг на размеченном корпусе.

### 5.4 Оператор кристаллизации

$$
\Kappa : \mathcal{F} \times \mathcal{I} \to \mathcal{F}
$$

подчиняется **четырём аксиомам безопасности**:

**(А1) Монотонность (Monotonicity).**
$$
\forall F, I :\ F \sqsubseteq \Kappa(F, I)
$$
Кристаллизация только расширяет мир; никогда не удаляет и не перезаписывает существующее содержимое.

**(А2) Ограниченная область (Bounded Domain).**
$$
\text{modified}(\Kappa(F, I)) \subseteq \mathcal{W}_{\text{crystalline}}
$$
где `$\mathcal{W}_{\text{crystalline}}$` — выделенная поддиректория `agents/{id}/.crystalline/`. Ядро проекта и `CLAUDE.md` неприкосновенны.

**(А3) Идемпотентность в локальном масштабе (Local Idempotence).**
$$
\Kappa(\Kappa(F, I), I) = \Kappa(F, I)
$$

**(А4) Аудируемость (Auditability).** Для каждой кристаллизации создаётся запись в `Karma Ledger`:
$$
L \mathrel{+}= \langle t, \{a_i\}, \pi(T_i), \Kappa\text{-diff}, \text{hash}(F_{\text{before}}), \text{hash}(F_{\text{after}}) \rangle
$$

### 5.5 Теорема о корректности

**Теорема (Интенциональная Безопасность).** Пусть выполняются аксиомы A1–A4. Тогда для любой последовательности чтений `$r_1, r_2, \dots, r_n$` в момент `$t$`:

$$
\text{Read}(r_j, F(t+\varepsilon)) \supseteq \text{Read}(r_j, F(t))
$$

т.е. ни один агент не может **потерять** данные, которые были бы ему доступны в отсутствие PSC. PSC может только **добавлять** контекст, и аддитивные кристаллизации образуют **решётку**, на которой состояние субстрата монотонно возрастает в пределах одного сеанса команды.

*Доказательство.* Следует непосредственно из A1 (монотонность) и A2 (ограниченность области) через индукцию по числу кристаллизаций. ∎

### 5.6 Оценка пропускной способности

Пусть `$f_{\text{think}}$` — средняя частота `thinking`-чанков на агента (эмпирически ≈ 2–4 Гц), `$k$` — число активных агентов, `$T_{\pi}$` — стоимость одного вызова проектора намерения. Стоимость PSC в секунду:

$$
C_{\text{PSC}} = k \cdot f_{\text{think}} \cdot T_{\pi} + f_{\Kappa} \cdot T_{\Kappa}
$$

При `$k = 2$`, `$f_{\text{think}} = 3$`, `$T_{\pi}^{\text{regex}} < 1$ мс`, `$T_{\pi}^{\text{slm}} \approx 150$ мс` (локальный 4B-параметрический Ollama-сервер на M4), стратегия `regex-first, slm-on-miss` даёт амортизированные ≈ 20 мс/сек — пренебрежимая доля CPU-бюджета Mac Mini M4.

### 6.2 Новый сервис: `ProlepticCrystallizer`

Файл: `apps/core/src/orchestrator/proleptic-crystallizer.service.ts`

```typescript
@Injectable()
export class ProlepticCrystallizer {
  private readonly buffers = new Map<string, ThinkingBuffer>();
  private readonly intentStore = new IntentStore();     // shared across agents
  private readonly ledger: KarmaLedger;

  constructor(
    private readonly projector: IntentProjector,
    private readonly aggregator: IntentAggregator,
    private readonly writer: CrystallineWriter,
    private readonly events: EventBusService,
  ) {}

  ingestThinking(agentId: string, text: string, ts: number): void {
    const buf = this.upsertBuffer(agentId);
    buf.append(text, ts);

    // Триггер проекции при накоплении ≥ 120 символов или паузе > 400 мс
    if (buf.shouldProject()) {
      const intent = this.projector.project(buf.drainWindow());
      if (intent.isNonEmpty()) {
        this.intentStore.upsert(agentId, intent);
        this.scheduleCrystallize(agentId);
      }
    }
  }

  sealIntent(agentId: string, toolUseChunk: StreamChunk): void {
    this.intentStore.seal(agentId, toolUseChunk);  // намерение подтверждено или отвергнуто
  }

  private scheduleCrystallize(triggerAgent: string): void {
    queueMicrotask(async () => {
      const aggregated = this.aggregator.aggregate(this.intentStore.all());
      const diff = await this.writer.crystallize(triggerAgent, aggregated);
      if (diff.nonEmpty) {
        this.ledger.record(diff);
        this.events.emit({ kind: 'psc.crystallized', content: diff.summary() });
      }
    });
  }
}
```

async projectViaSlm(text: string): Promise<RawIntent[]> {
  const prompt = `From the reasoning fragment below, extract pending filesystem operations as JSON array with fields {path, op, hint}. Output JSON only.\n\n---\n${text}\n---`;
  const res = await this.ollama.generate({

### 6.4 `IntentAggregator`

Файл: `apps/core/src/orchestrator/intent-aggregator.ts`

Реализует оператор `⊕` из §5.2. Дополнительно обнаруживает:
- **Коллизии намерений**: два агента намерены писать в один путь → помечается флаг `conflict`, поддиректория расщепляется на `forks/{agentId}/`.
- **Резонансы намерений**: ≥ 2 агентов намерены читать один путь → вес кристаллизации поднимается; создаётся `shared.context.md` в соответствующей директории.

### 6.5 `CrystallineWriter`

Файл: `apps/core/src/orchestrator/crystalline-writer.ts`

Реализует оператор `Κ` из §5.4. Пишет **исключительно** в `data/workspaces/{teamId}/agents/{agentId}/.crystalline/`. Формы кристаллизации:

| Форма | Триггер | Содержимое |
|---|---|---|
| `hint-{n}.md` | READ-интенция на файл X в проекте | Краткое резюме файла X (из локального кэша) + ссылки на связанные файлы |
| `peer-intent.md` | RESO на путь P между двумя агентами | «Агент `agent_Xyz` также намеревается читать P для задачи Y. Рассмотрите координацию.» |
| `forks/...` | Конфликт записей | Изолированные версии для каждого претендующего агента |
| `scratch-{path}.md` | WRITE-интенция на несуществующий файл | Пустой шаблон + метаданные |

Агент в своём CLAUDE.md получает строку: `*The directory `.crystalline/` may contain proactive hints compiled from the team's current work. Read its index before heavy tool use.*`

### 6.9 Метрики эксперимента

Для валидации эффекта PSC вводится A/B-режим (`env PSC_ENABLED=false`). Метрики:

- `turns_to_completion` — среднее число ходов агента до `mark-done`.
- `redundant_reads` — процент чтений, возвращающих то же содержимое, что уже было прочитано соседним агентом в текущем сеансе команды.
- `intent_hit_rate` — доля спроецированных интенций, подтверждённых последующим `tool_use`.
- `crystallization_utility` — доля кристаллизованных файлов, фактически прочитанных агентом.

Гипотеза: `turns_to_completion` снизится на ≥ 15%, `redundant_reads` — на ≥ 40% при `intent_hit_rate ≥ 0.55`.

## VII. Эпистемологическое Замечание

PSC реализует тонкую инверсию стандартной причинной схемы мультиагентных систем. Классическая MAS: **действие → среда → восприятие → следующее действие**. PSC вводит параллельный причинный путь: **намерение → среда → восприятие → действие**, где «намерение» — это физически существующая сущность в NDJSON-потоке, а не выведенная модель.

Можно сказать, что PSC трактует `thinking`-чанки как **физические наблюдаемые величины**, из которых оркестратор добывает работу (через кристаллизацию субстрата), не нарушая когнитивную автономию агента — ибо агент никогда не обнаруживает источник обогащённого контекста. Это имеет философский резонанс с слабым измерением в квантовой механике (Aharonov, Albert, Vaidman, 1988), где извлекается информация без полного коллапса состояния, — но здесь применяется к эпистемической, а не квантовой наблюдаемости.

## VIII. Квалификация

PSC удовлетворяет критериям нулевого предшествующего уровня техники:

| Критерий | Результат |
|---|---|
| Присутствует ли в существующих MAS-фреймворках? | Нет (структурная невозможность). |
| Покрывается ли публикациями arXiv/IEEE/ACM? | Отдельные компоненты (стигмергия, ToM, world models) покрыты; их **конъюнкция как новый класс согласованности** — не обнаружена. |
| Применима ли к системам на внешних LLM API? | Нет. Требует синхронного доступа к `thinking` до `tool_use`. |
| Является ли вариацией ВТМ (первого цикла)? | Нет. Ортогональная ось: ВТМ — наблюдение одиночного потока; PSC — актуация субстрата из коллективных потоков. |
| Может ли быть реализована за 2–3 недели инженерного труда? | Да. |

**Итоговая квалификация.** Проспективная Кристаллизация Субстрата — **структурно-привязанный оркестрационный примитив**, порождающий новый класс согласованности **Intentional Consistency**. Примитив реализуем исключительно в конъюнкции локального запуска LLM, синхронного доступа к потоку мышления, кросс-агентного наблюдения оркестратором и разделяемого файлового субстрата. Нарушение любого из четырёх условий редуцирует концепт к уже известному паттерну. Соответственно, PSC представляет собой подлинное открытие, достижимое исключительно в архитектуре Gamma Runtime v2.
