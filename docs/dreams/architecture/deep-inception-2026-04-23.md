## Сводка саб-агентов (Хаос-лог)

### Саб-агент Альфа — Биохакер (Документы 1 + 2)

Альфа реинтерпретировал архитектуру Gamma OS как биологический организм. Жизненный цикл агента — это клеточный метаболизм; SessionPool — иммунная система; SSE-поток событий — нейронная ткань. Пять выдвинутых механизмов:

- **Метаболический градиент давления**: при достижении 70% заполненности контекстного окна агент «переваривает» холодную память — сжимает завершённые выводы инструментов и итоги задач — вместо полного сброса сессии. При критическом пороге происходит «митоз»: задача расщепляется, дочерний агент наследует сжатую половину контекста. Аналог клеточного аутофага под нутриентным стрессом.

- **Проприоцептивное управление порождением**: в XR-интерфейсе продолжительность фиксации взгляда оператора на узле агента управляет его приоритетом в очереди FIFO. Жест pinch длительностью >800 мс активирует «адреналиновый режим» — временное увеличение `MAX_CONCURRENT_AGENTS` на единицу сроком на 90 секунд с последующей жёсткой фиксацией. Аналог симпатической активации нервной системы.

- **Маршрутизация через нейронные спайк-тренды**: SSE-события от нескольких агентов кросс-коррелируются в реальном времени. Совпадение событий `agent.thinking` в окне 200 мс по одному домену генерирует синтетическое событие `cortex.resonance` — сигнал о функциональном сопряжении агентов. Аналог hebbian-синхронии.

- **Апоптотический каскад тайм-аута**: ошибка агента генерирует событие `stress.neighbor_failed` для соседних агентов команды. При трёхкратном накоплении лидер-агент принудительно пробуждается с инъекцией «гормонального» префикса в промпт. Аналог химической сигнализации апоптоза соседними клетками.

- **Циркадный цикл простоя**: агенты в состоянии `idle` дольше 10 минут входят в фазу «медленноволновой консолидации» — последние 5 трейс-событий воспроизводятся как сжатый внутренний монолог при следующем запуске. После 60 минут простоя предварительно компилируется diff рабочего пространства для минимизации задержки пробуждения.

### Саб-агент Бета — Квантовый Экономист (Документы 2 + 3)

Бета спроецировал архитектуру на принципы информационной экономики. Поведения агентов, валидированные знания и паттерны взаимодействия становятся ликвидными активами с присущей им волатильностью и механизмами ценообразования.

- **Облигации с периодом полураспада навыков**: каждая запись в `shared/skills/` несёт коэффициент деградации, снижающийся по мере накопления контрсвидетельств в `shared/anomalies.md`. Агенты, опирающиеся на устаревшее знание, «платят» больше контекстных токенов за каждое обращение — система органически вытесняет устаревшие знания без явного устаревания.

- **Короткие позиции по аномалиям**: отклонённые гипотезы — это шорт-контракты. Агент, воспроизводящий провальный подход в новом контексте, либо «закрывает шорт» (провал был контекстно-специфичным), либо углубляет его (провал универсален). Закрытие шорта приносит дивиденды доверия: будущие предложения навыков агента получают повышенный начальный рейтинг.

- **Ликвидные премии за беглость взаимодействия**: паттерны взаимодействия, достигшие порога беглости (задержка <150 мс, отсутствие коррекций motion sickness), маркируются как «ликвидные». Критические подтверждения операций маршрутизируются преимущественно через ликвидные паттерны; новые жесты несут «налог трения».

- **Арбитражные окна составных эффектов**: при обнаружении суперадддитивного взаимодействия двух навыков открывается ограниченное по времени арбитражное окно. Первые агенты, эксплуатирующие составной эффект, получают дивиденды в виде сниженной стоимости конкуренции за слоты в `SessionPool`. Окно закрывается после N прогонов команды — когда инсайт становится общеизвестным.

- **Долг присутствия**: пространственные прерывания, нарушающие эргономические пороги (вторжение в периферийный AR-оверлей во время активного gaze-взаимодействия), накапливают долг по сессии. Агенты-нарушители получают штрафной мультипликатор задержки пробуждения при следующем запуске.

### Саб-агент Гамма — Хаос-Инженер (Синтез Альфа + Бета, применение энтропии)

Гамма вскрыл скрытое общее допущение Альфы и Беты: **оба предполагают достоверность сигнала под нагрузкой.** Под высокой конкурентностью точное временное окно (200 мс для корреляции, <150 мс для ликвидности) становится зашумлённым. Это порождает положительную петлю обратной связи: апоптотические каскады стимулируют порождение агентов → увеличение плотности SSE-событий → разрушение тайминговых окон → ложные `cortex.resonance` → новые порождения → исчерпание конкурентного бюджета → жёсткая фиксация → уничтожение активных агентов → каскад ошибок. Система пожирает себя менее чем за 90 секунд. Одновременно долг присутствия откладывает пробуждения именно тогда, когда оркестратор наиболее нуждается в рабочей силе, а облигации с полураспадом увеличивают стоимость токенов в момент наивысшего контекстного давления.

Оба предложенных механизма оптимальны в спокойных условиях и катастрофичны под той нагрузкой, для управления которой они были разработаны.

Решение — не усиленное демпфирование, а **намеренная инъекция шума в состоянии покоя**. Система, уже функционирующая вблизи границы хаоса, не опрокидывается катастрофически под нагрузкой — она уже адаптирована к турбулентности. Спокойные периоды следует использовать для мягкого самовозмущения, наращивая структурную устойчивость до прихода давления.

Единый паттерн: **Strange Attractor Mesh (SAM)** — оркестратор поддерживает плавающий индекс турбулентности (0.0–1.0), обновляемый каждые 2 секунды. При турбулентности <0.3 система намеренно инжектирует хаос. При турбулентности >0.7 все вторичные сигналы подавляются, маршрутизация сводится к примитивному FSM (assign/complete/fail). Аттрактор — диапазон 0.3–0.7. Визуальный признак здоровья системы — шум.

## Мета-Синтез (Финальный концепт)

### Живая Оркестровая Ткань (ЖОТ) / *Living Orchestral Fabric*

Из трёх саб-агентов извлекается единый мета-принцип: **Gamma OS должна функционировать как самоорганизующаяся диссипативная структура** — система, поддерживающая собственную сложность за счёт непрерывного потребления и рассеивания энергии, никогда не достигающая истинного равновесия. Подобные структуры описаны Пригожиным (Prigogine, 1977) как фундаментальный механизм живой организации; Лэнгтон (Langton, 1990) показал, что на краю хаоса вычислительная ёмкость сложных адаптивных систем максимальна.

**Постулат I. Агент как диссипативная ячейка.**
Каждый агент — не конечный автомат с дискретными состояниями `idle/running/error`, а точка в четырёхмерном фазовом пространстве:
- **σ** (контекстное насыщение) — отношение текущих токенов к `context_window`, [0..1]
- **τ** (темп задач) — скользящее среднее завершённых задач в единицу времени, нормализованное
- **κ** (уверенность в знаниях) — взвешенная сумма confidence-коэффициентов применённых навыков
- **ε** (вклад в турбулентность) — мера того, насколько события агента дестабилизируют EventBus

Назначение задач осуществляется не по статическому сопоставлению ролей, а по косинусному расстоянию в этом фазовом пространстве между вектором требований задачи и вектором состояния агента.

**Постулат II. EventBus как нейронная ткань с временно́й памятью.**
Шина событий не просто ретранслирует — она поддерживает скользящее корреляционное окно (30 секунд) над всеми событиями. Паттерны синхронности вычисляются непрерывно. EventBus трансформируется из пассивного транспорта в активный детектор функционального связывания агентов.

**Постулат III. Знание как радиоактивный материал.**
Каждая запись в базе знаний несёт период полураспада, определяемый доменом (технические детали — короткий период, архитектурные принципы — длинный) и скоростью накопления контрсвидетельств. Когда суммарная «активность» применяемых знаний опускается ниже критической отметки, система автоматически генерирует задачу повторной валидации — знание должно зарабатывать своё существование в каждом новом поколении.

**Постулат IV. Странный аттрактор как принцип управления.**
Gamma OS не оптимизирует производительность — она удерживает себя в диапазоне турбулентности 0.3–0.7. Ниже нижней границы система слишком детерминирована: не адаптируется к новым условиям. Выше верхней — хаотична и ненадёжна. В диапазоне — максимально креативна и отказоустойчива. Индикатором здоровья системы является характер шума: в состоянии покоя TaskBoard изредка самопроизвольно переводит задачи в `planning`, два простаивающих агента обмениваются загадочными сообщениями, показатель уверенности навыка снижается без видимой причины. Под нагрузкой всё это прекращается. Когда буря проходит — начинается снова.

**Постулат V. Пространственный интерфейс как моторная кора.**
XR-интерфейс — не пассивная визуализация состояния системы, а орган управления с собственной моторной памятью. Беглые жесты (gaze+pinch, выдержанный взгляд) имеют прямые привилегии оркестрации: они воздействуют непосредственно на индекс турбулентности и очередь `SessionPool`, минуя REST API. Интерфейс и система формируют замкнутый контур обратной связи, аналогичный соматосенсорно-моторной петле в нейронауках.

## Схема имплементации в Gamma OS

### Модуль 1: `TurbulenceService` — вычисление индекса

```typescript
@Injectable()
export class TurbulenceService implements OnModuleInit {
  private index = 0.5; // стартовое значение в центре аттрактора
  private readonly eventBuffer: number[] = []; // кольцевой буфер временных меток
  private readonly BUFFER_SIZE = 90; // ~30 сек при ~3 событиях/сек

  onModuleInit() {
    this.eventBus.onAny(() => {
      this.eventBuffer.push(Date.now());
      if (this.eventBuffer.length > this.BUFFER_SIZE) this.eventBuffer.shift();
    });
    setInterval(() => this.recompute(), 2000);
  }

  private recompute() {
    if (this.eventBuffer.length < 10) return;
    const intervals = this.eventBuffer.slice(1).map((t, i) => t - this.eventBuffer[i]);
    const mean = intervals.reduce((a, b) => a + b, 0) / intervals.length;
    const variance = intervals.reduce((a, b) => a + (b - mean) ** 2, 0) / intervals.length;
    // Коэффициент вариации — безразмерный индекс турбулентности
    this.index = Math.min(1, Math.max(0, Math.sqrt(variance) / mean / 3));
  }

  get isCalm() { return this.index < 0.3; }
  get isStorm() { return this.index > 0.7; }
  get value() { return this.index; }
}
```

@Injectable()
export class ChaosInjector {
  private lastInjectionAt = 0;
  private readonly MIN_INTERVAL_MS = 10_000;

  async maybeInject(teamId: string): Promise<void> {
    if (!this.turbulence.isCalm) return;
    if (Date.now() - this.lastInjectionAt < this.MIN_INTERVAL_MS) return;

    const perturbations = [
      () => this.knowledgeService.decayRandomSkill(teamId),
      () => this.replayRandomAnomaly(teamId),
      () => this.forcePeerMessage(teamId),
      () => this.markOldestBacklogStale(teamId),
    ];

    const chosen = perturbations[Math.floor(Math.random() * perturbations.length)];
    await chosen();
    this.lastInjectionAt = Date.now();
    this.eventBus.emit('chaos.injection', { teamId, turbulence: this.turbulence.value });
  }
}

### Модуль 4: `PhaseSpaceRouter` — маршрутизация через фазовое пространство

```typescript
interface AgentPhaseVector {
  agentId: string;
  sigma: number;   // context saturation [0..1]
  tau: number;     // task velocity, normalized
  kappa: number;   // knowledge confidence [0..1]
  epsilon: number; // turbulence contribution [0..1]
}

@Injectable()
export class PhaseSpaceRouter {
  private vectors = new Map<string, AgentPhaseVector>();

  cosineSimilarity(a: AgentPhaseVector, b: AgentPhaseVector): number {
    const dot = a.sigma * b.sigma + a.tau * b.tau + a.kappa * b.kappa + a.epsilon * b.epsilon;
    const magA = Math.sqrt(a.sigma ** 2 + a.tau ** 2 + a.kappa ** 2 + a.epsilon ** 2);
    const magB = Math.sqrt(b.sigma ** 2 + b.tau ** 2 + b.kappa ** 2 + b.epsilon ** 2);
    return magA && magB ? dot / (magA * magB) : 0;
  }

  // Найти агента, ближайшего к целевому вектору задачи в фазовом пространстве
  findOptimalAgent(taskProfile: Partial<AgentPhaseVector>): string | null {
    const target = { sigma: 0, tau: 0.5, kappa: 1, epsilon: 0, ...taskProfile } as AgentPhaseVector;
    let bestId: string | null = null;
    let bestScore = -Infinity;
    for (const [id, vec] of this.vectors) {
      const score = this.cosineSimilarity(vec, target);
      if (score > bestScore) { bestScore = score; bestId = id; }
    }
    return bestId;
  }
}
```

### Интеграция с локальными LLM (M4 Neural Engine)

В будущих версиях `TurbulenceService` может делегировать вычисление индекса локальной языковой модели (через Ollama или llama.cpp), выполняемой на M4 Neural Engine. Модель получает последние N трейс-событий в формате JSON и возвращает семантическую оценку турбулентности с кратким обоснованием — переводя метрику из статистической плоскости в смысловую.

Семантический индекс турбулентности способен различать «агенты конкурируют из-за взаимоисключающих подходов к одному модулю» и «агенты параллельно решают независимые задачи» — различие, принципиально недоступное чисто статистическим методам. Это соответствует духу проекта: агенты уже используют локально запущенный Claude CLI; добавление специализированной наблюдательной модели — органическое расширение, не нарушающее принципа local-first.
